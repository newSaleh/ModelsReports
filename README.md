# تقرير المبيعات الأسبوعية — جميع الفروع

تطبيق ويب لتحليل تقرير المبيعات الأسبوعي لكل فرع، مع إمكانية تعديل البيانات مباشرة،
وتصدير التقرير كملف Excel أو PDF مستقل لكل فرع.

يعمل بالكامل من داخل المتصفح (بدون سيرفر أو قاعدة بيانات) ويمكن استضافته مباشرة عبر GitHub Pages.
**لا تُشحن أي بيانات حقيقية مع الكود** — التطبيق يبدأ فارغًا، وتقوم برفع ملف Excel الأسبوعي بنفسك.

|---|---|
| 701 | الدائري |
| 706 | الفيحاء |

| 711 | بريدة |
| 803 | التحلية |

## الميزات

- **تعديل مباشر**: كل خلية في جدول البيانات (المورد، الفئة، الأسعار، الكميات المباعة، الأرصدة لكل فرع) قابلة للتعديل، ويُحفظ كل تغيير تلقائيًا في متصفحك (localStorage).
- **استيراد Excel**: رفع ملف أسبوعي بنفس هيكلة الأعمدة (زر "نموذج فارغ" ينزّل نموذجًا بالأعمدة الصحيحة).
- **تصدير Excel**: تصدير البيانات الحالية كملف xlsx.
- **تصدير PDF لكل فرع على حدة**: اختر فرعًا من القائمة المنسدلة "تقرير فرع" ثم اضغط "تصدير PDF" لإنشاء
  تقرير خاص بذلك الفرع فقط، أو اختر "كل الفروع" لإنشاء تقرير شامل (صفحة ملخص + تقرير مستقل لكل فرع).
  التقرير مبني من نفس واجهة المتصفح مباشرة (عبر html2canvas) فيظهر النص العربي بشكله الصحيح
  تمامًا كما يظهر على الشاشة، بدون أي قص أو تشويه.
- **تقرير كل فرع يعرض لكل صنف**: الفئة والموديل، السعر، الكمية المباعة في هذا الفرع تحديدًا،
  الكمية المباعة إجمالًا في باقي الفروع، الرصيد الحالي بالفرع، وحالة واضحة:
  - 🔴 **لا يوجد رصيد — اطلب الآن**: الصنف يبيع جيدًا (في هذا الفرع أو في باقي الفروع) ولا يوجد منه رصيد إطلاقًا.
  - 🟡 **رصيد منخفض — اطلب الآن**: نفس الحالة لكن يوجد رصيد قليل جدًا (أقل من أسبوع تغطية).
  - 🟢 **موديل ناجح غير متوفر لديك**: صنف ناجح جدًا في باقي الفروع لكنه غائب تمامًا عن هذا الفرع.
  - **المخزون مناسب**: بقية الأصناف.

  التقرير **لا يقترح نقل بضاعة بين الفروع** — فقط يقيّم كل فرع بمفرده بناءً على مبيعاته ورصيده،
  ويستخدم أداء الصنف في باقي الفروع كدليل إضافي على أنه يستحق طلب توريد.
- **لوحة تحليل**: أفضل 10 موديلات مبيعًا (للفرع المختار أو لكل الفروع)، وإجمالي المبيعات حسب الفرع.

## طريقة الاستخدام

### محليًا
افتح `index.html` مباشرة في المتصفح، أو شغّل خادمًا محليًا بسيطًا:

```bash
python3 -m http.server 8000
# ثم افتح http://localhost:8000
```

### عبر GitHub Pages
من إعدادات المستودع على GitHub: Settings → Pages → اختر الفرع الذي يحتوي هذا الكود
كمصدر (Source) والمجلد الجذر (`/`). سيصبح التطبيق متاحًا عبر رابط Pages خلال دقائق.

SupplierCode, SupplierName, StockGroupName, StockCode, ModelCode, UnitPrice,

## ضبط حساسية التقييم

يمكن تعديل الحدود من أعلى ملف `assets/app.js`:

- `HOT_SOLD_MIN`: الحد الأدنى للمبيعات الأسبوعية في الفرع نفسه ليُعتبر الصنف "يبيع جيدًا" (افتراضي 5).
- `CRITICAL_WEEKS_LEFT`: عدد أسابيع التغطية التي يعتبر أقل منها الصنف بحاجة لطلب فوري (افتراضي 1).
- `OPPORTUNITY_MIN_TOTAL_SOLD`: الحد الأدنى لإجمالي المبيعات في باقي الفروع ليُعتبر الموديل "ناجحًا" يستحق تزويد فرع جديد به.
